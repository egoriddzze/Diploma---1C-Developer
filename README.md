# Инструкция по начальной настройке

* Все перечисленные справочники, документы и регистры в инструкции, доступны в подсистеме «ВКМ_ДобавленныеОбъекты».

Шаг 1

Для начала вам нужно создать группу и бота в Telegram. Это потребуется для оповещения специалистов о созданных документах «Обслуживание клиента».

Шаг 2

Заполните следующие константы:

*«Номенклатура абонентская плата» - заполняется из справочника «Номенклатура» подходящим по смыслу наименованием, при необходимости создайте новую номенклатуру. 

*«Номенклатура работы специалиста» - заполняется из справочника «Номенклатура» подходящим по смыслу наименованием, при необходимости создайте новую номенклатуру. 

«Идентификатор группы ТГ» - заполняется идентификатором группы Telegram. 

«Токен управления телеграм ботом» - заполняется токеном бота Telegram

*Если хоть одна из данных констант не будет заполнена, то документ «Реализация товаров и услуг» при нажатии на кнопку «Заполнить», заполнен не будет!

Шаг 3

Заполните регистр «Условия оплаты сотрудников». Для каждого сотрудника укажите: оклад и процент от выполненных работ, а так же категорию сотрудника. Это понадобится для корректной работы расчетных механизмов.

Шаг 4

В справочнике «Режимы графиком работы» заведите необходимый режим графика(например, пятидневка). Далее откройте обработку «Заполнение графика». В поле выбора периода выберите период для заполнения графика. В поле «Выходные дни» укажите через запятую порядковый номер выходного дня недели(например: 6 для субботы и 7 для воскресенья). В поле «График работы» выберите ранее созданный вами элемент справочника «Режимы графиком работы». Нажмите на кнопку «Заполнить». После этого откройте регистр «Графики работы» и проверьте корректность заполненных данных.

Шаг 5

В справочнике «Пользователи» назначьте для пользователей необходимые новые группы доступа. Все новые профили групп доступа являются поставляемыми и не могут быть изменены пользователем самостоятельно. Подробнее с составом ролей профилей групп доступа вы можете ознакомиться по следующему пути: «Администрирование» -> «Настройки пользователей и прав» -> «Профили групп доступа» -> папка «ВКМ».

Шаг 6

Проинструктируйте бухгалтера о возможности автоматического создания документов по реализации за месяц с помощью обработки «Массовое создание актов».

Шаг 7

Проинструктируйте менеджера о возможности создания договоров вида «Абонентское обслуживание». Для них доступно заполнение дополнительных полей: «дата начала и окончания договора», «стоимость часа работы» и «абонентская плата». Все эти поля должны быть заполнены для корректного формирования документов реализации.

# Инструкция по настройке автоматизированного тестирования

Состав сценариев тестирования:

1.МенеджерСозданиеОбслуживаниеКлиента.
2.СпециалистПроведениеДокументаОбслуживаниеКлиента.
3.БухгалтерМассовоеСозданиеДокументов.
4.БухгалтерОтчетАнализВыставленныхАктов.

Порядок действий тестирования

1. Скачать внешнюю обработку vanessa-automation-single.epf.
2. Создать новую пустую базу для менеджера тестирования(в ней будет производиться запуск обработки для тестирования).
3. В дополнительных параметрах запуска базы для менеджера тестирования пропишите /TESTMANAGER.
4. Выгрузить dt рабочей базы.
5. Добавить новую базу для клиента тестирования.
6. В базу для клиента тестирования загрузить dt рабочей базы.
7. В конфигураторе базы для клиента тестирования снимите флаг «Защита от опасных действий» у пользователей Менеджер, Специалист, Бухгалтер.
8. Запустите базу для клиента тестирования, при запуске укажите, что база перемещена(это копия базы).
9. В дополнительных параметрах запуска базы для клиента тестирования пропишите /TestManager.
10. Запустите базу для клиента тестирования под пользователем Администратор.
11. Откройте внешнюю обработку vanessa-automation-single.epf.
12. После запуска обработки перейдите на вкладку Клиенты тестирования, убедитесь, что тестовая база прописана как клиент тестирования. При необходимости пропишите её.

Тестирование:

1. Нажмите кнопку Загрузить фичи.
2. Укажите путь к файлам сценариев.
3. Выберите сценарий. Сценарии тестирования следует запускать в следующей очерёдности: МенеджерСозданиеОбслуживаниеКлиента от имени Менеджера-> СпециалистПроведениеДокументаОбслуживаниеКлиента от имени Специалиста-> БухгалтерМассовоеСозданиеДокументов от имени Бухгалтера-> БухгалтерОтчетАнализВыставленныхАктов от имени Бухгалтера.
4. Для каждого сценария, в настройках клиента тестирования необходимо прописывать пользователя, от имени которого будет запущен сценарий.
5. Запустите выбранный сценарий.
6. Должен запуститься клиент тестирования от имени пользователя, который указан в настройках клиента тестирования сценарии.
7. Следите за выполнением сценария на клиенте тестирования.
8. После завершения сценария вы получите звуковое и оповещение о завершении сценария. Сам сценарий окрасится в зеленый цвет.
